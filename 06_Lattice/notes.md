A lattice is just a grid of dots.

In 2D it looks exactly like graph paper:

```
· · · · · · ·
· · · · · · ·
· · · · · · ·
· · · · · · ·
· · · · · · ·
```

Every dot is a point. The grid extends infinitely in all directions. That is a lattice.

Now imagine the same thing but in 256 dimensions instead of 2. You cannot draw it. But the math works exactly the same way.

---

**What makes a lattice mathematical:**

A lattice is not just any random grid. It is generated by a set of basis vectors.

Think of it like directions:

```
Vector 1 = go 2 steps right, 1 step up
Vector 2 = go 0 steps right, 3 steps up
```

Every point in the lattice is reached by combining these vectors:

```
Point = (a × Vector1) + (b × Vector2)
```

Where `a` and `b` are any whole numbers — positive, negative, or zero.

So the lattice is completely defined by its basis vectors. Change the basis vectors, you get a different lattice.

---

**This is where the cryptography lives.**

The same lattice can be described by two completely different sets of basis vectors:

```
Good basis = short, clean vectors  → easy to work with
Bad basis  = long, messy vectors   → hard to work with
```

Both describe the exact same lattice. Same dots. Same grid. But one basis makes problems easy and the other makes them hard.

The private key in Dilithium is essentially the good basis. The public key is the bad basis. Everyone can see the bad basis. Nobody can find the good basis from it.

---

**Now the hard problems. There are three main ones.**

---

**Problem 1 — SVP — Shortest Vector Problem**

Given a lattice described with a messy bad basis — find the shortest possible distance between any two dots.

In 2D this sounds trivial. You can just look at the graph paper and see it.

In 256 dimensions you cannot see anything. The number of possible vectors to check is astronomically large. Even quantum computers cannot solve this efficiently when the dimension is high enough.

This is the foundational hard problem. Everything else builds on this.

```
Simple version:
"Given this horrible messy description of a grid
 find the two dots that are closest to each other"

Easy in 2D → look and see
Impossible in 256D → no algorithm can do it fast
```

---

**Problem 2 — CVP — Closest Vector Problem**

Given a lattice and a random target point that is NOT on the lattice — find the lattice point closest to that target.

The target point is floating somewhere in space. It does not sit on any dot of the grid. You need to find which dot is nearest to it.

```
Simple version:
"Here is the grid
 Here is a random point floating nearby
 Which dot is it closest to?"

Easy in 2D → look and see
Impossible in 256D → computationally infeasible
```

CVP is actually harder than SVP in general. If you can solve SVP you can sometimes solve CVP. But CVP is the more practically relevant problem in cryptography.

This is what an attacker needs to solve to break lattice-based encryption. They have the ciphertext — a point floating near the lattice — and they need to find the secret lattice point it came from.

---

**Problem 3 — LWE — Learning With Errors**

This one is the most important for modern PQC. Dilithium and Kyber are both based on LWE.

Start with a simple equation:

```
b = a × s (mod q)
```

Where `a` is public, `s` is the secret, `q` is a modulus. If you see enough `(a, b)` pairs you can solve for `s` easily — it is just linear algebra.

Now add a tiny random error `e` to every equation:

```
b = a × s + e (mod q)
```

The error is small. But it completely destroys your ability to solve for `s` using linear algebra. The system of equations is now slightly wrong in a random way that you cannot separate from the real signal.

```
Without error → easy to solve → insecure
With error    → impossible to solve → secure
```

This is Learning With Errors. You are trying to "learn" the secret `s` but the errors prevent you from doing so.

---

**How LWE connects to lattices:**

Solving LWE is mathematically equivalent to solving CVP on a certain type of lattice. So the hardness of LWE inherits directly from the hardness of CVP. They are the same problem wearing different clothes.

This connection is what gives LWE its security proof. NIST accepted Dilithium and Kyber specifically because their security reduces to LWE which reduces to lattice problems which have been studied for decades with no known quantum algorithm to break them.

---

**Problem 4 — RLWE — Ring Learning With Errors**

LWE is secure but slow. The keys and operations involve large matrices which take a lot of memory and computation.

RLWE is LWE but performed inside a mathematical structure called a polynomial ring. Instead of matrices you work with polynomials.

```
LWE  → matrices → large keys → slow
RLWE → polynomials → compact keys → fast
```

This is what Dilithium and Kyber actually use. RLWE gives you the same security as LWE but with much smaller keys and faster operations. That is why they were practical enough for NIST to standardize.

---

**Why quantum computers cannot solve these problems:**

Shor's Algorithm works by finding periods in modular exponentiation sequences. It exploits the specific mathematical structure of integer factorization and discrete logarithm.

Lattice problems have no such periodic structure to exploit. Shor's shows up to SVP or CVP or LWE and has nothing to grab onto. There is no Fourier Transform trick that simplifies these problems. Even Grover's Algorithm — the other major quantum algorithm — only gives a tiny speedup against lattice problems, not enough to break them at practical key sizes.

This is not just an assumption. Lattice problems have been studied intensively since the 1980s by both classical and quantum researchers. Nobody has found an efficient algorithm. NIST spent 7 years evaluating submissions and specifically chose lattice-based schemes because of this track record.

---

**The full picture in one table:**

| Problem | Simple meaning                         | Used in                           |
| ------- | -------------------------------------- | --------------------------------- |
| SVP     | Find shortest vector in lattice        | Foundation of all lattice crypto  |
| CVP     | Find closest lattice point to a target | Encryption schemes                |
| LWE     | Solve equations with random errors     | Dilithium, Kyber                  |
| RLWE    | Same as LWE but with polynomials       | Dilithium, Kyber (faster version) |

---

**One paragraph summary:**

Lattice cryptography hides secrets inside high-dimensional grids where finding anything is computationally impossible. The good basis is your private key. The bad basis is your public key. Attacking the system means solving SVP or CVP or LWE — problems that have resisted every classical and quantum attack for decades. Shor's Algorithm is completely blind to them because they have no periodic structure to exploit. That is why Dilithium and Kyber are quantum resistant. Not by assumption. By decades of mathematical evidence.
